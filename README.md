# ChatGPTCodexSync

Portable Windows utility for backing up and restoring the `%USERPROFILE%\.codex` folder used by ChatGPT Desktop / Codex.

The application is designed to run from a folder synchronized by tools such as Nextcloud. It does not integrate with Nextcloud directly; backups are written to the local application folder and can then be synchronized by the existing sync client.

## Current Phase

The project is currently in the restore foundation phase.

Completed so far:

- WPF application shell with MVVM structure.
- Dependency Injection, configuration, and logging setup.
- Portable application icon and version display.
- MIT license.
- Initial backup workflow for `%USERPROFILE%\.codex`.
- Backup manifest generation.
- ZIP fallback support.
- Portable 7-Zip support with automatic tool preparation.
- Fast, Balanced, and Maximum 7-Zip compression profiles aligned with the proven `SaveCodex.cmd` style.
- Initial safe restore workflow for ZIP and 7z backups.
- Backup manifest validation during restore.
- Temporary extraction before touching the current profile.
- Safety backup creation before replacing the current `.codex` directory.
- Persistent operation logs written to the executable `Logs` directory.
- Selectable operation log verbosity: Normal, Detailed, and Diagnostic.
- Backup and restore progress now starts from 0% for long-running archive phases.
- Real-time log output, progress percentage, and auto-scrolling log view.
- GitHub Actions workflow for Windows x64 release builds.
- Initial automated tests.

## Releases

GitHub Releases are generated by `.github/workflows/release.yml` when a version tag is pushed.

Release titles and notes must be written in English and should describe the actual changes included in the tagged version.

Example:

```text
git tag v0.2.3
git push origin v0.2.3
```

The workflow builds, tests, publishes the self-contained Windows x64 application, and attaches a ZIP archive to the GitHub Release.
